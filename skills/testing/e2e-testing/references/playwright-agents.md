## Origin

Playwright Test Agents shipped in **Playwright 1.56**, October 2025.
They sit on top of the official `@playwright/mcp` server (Microsoft).
The agents are AI-driven — they call MCP tools to interact with a real
browser, not a stub.

## The three agents

### Planner

- **Input** — a live URL, plus a free-form description of what to explore.
- **Action** — opens the app, walks the visible features via the
  accessibility tree, and writes a structured Markdown plan.
- **Output** — `specs/<area>/<flow>.md` (human-readable, version-controlled).
- **Use when** — the feature already exists in the app and no spec captures
  it yet.

### Generator

- **Input** — a Markdown spec produced by the Planner or by the user.
- **Action** — opens the live app, walks the spec step by step, picks
  locators from the accessibility tree, writes a Playwright test.
- **Output** — `tests/<area>/<flow>.spec.ts`.
- **Constraints** — uses the locator ladder
  (see [`rules/locator-strategy.md`](../rules/locator-strategy.md)),
  reuses `storageState`, one assertion per spec assertion.

### Healer

- **Input** — a failing test plus its trace.
- **Action** — opens the live app, snapshots the page, finds the equivalent
  element via the accessibility tree, patches the locator, re-runs.
- **Output** — patched test (rungs 1–2), or a `data-testid` source-diff
  proposal (rung 3).
- **Constraint** — three-attempt cap per failing test
  (see [`rules/token-budget.md`](../rules/token-budget.md)).

Microsoft's reported success rate on selector-related failures: 75%+.

## Setup

Initialise agent definitions for Claude Code:

```bash
npx playwright init-agents --loop=claude
```

This generates the agent definitions and the recommended folder layout.
Re-run after every Playwright minor upgrade — definitions evolve.

Verify `@playwright/mcp` is installed:

```bash
npm i -D @playwright/mcp@latest
```

Add the MCP server to the Claude Code MCP config (project or user scope):

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    }
  }
}
```

## Folder layout produced

```
repo/
├── specs/                     # Markdown plans, owned by the Planner / user
├── tests/
│   ├── seed.spec.ts           # auth + storageState bootstrap
│   └── <area>/<flow>.spec.ts  # generated by the Generator
├── playwright.config.ts
└── .github/                   # agent definitions written by init-agents
```

`specs/` and `tests/` are version-controlled.
`storageState.json` is **not** version-controlled — it contains live cookies.

## Invocation patterns

### Planner

```
Planner: explore http://localhost:3000/dashboard and write a spec for
"create project" into specs/dashboard/create-project.md.
```

### Generator

```
Generator: generate tests/dashboard/create-project.spec.ts from
specs/dashboard/create-project.md.
```

### Healer

```
Healer: tests/dashboard/create-project.spec.ts is failing on the
"create project" assertion — inspect the page, propose a fix.
```
