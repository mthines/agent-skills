# Report templates

Two variants — pick by mode.
Write the file to `stabilization-report.md` (stabilize) or `optimization-report.md` (optimize) at the repo root or worktree root.
Render markdown; do not collapse into prose.

---

## Residual risk

Anything still hot:

- Tests in the fix queue that are now `improved` rather than `fixed` — failure_rate non-zero but reduced.
- Tests marked `requires-human-judgment` (hypothesis did not map to a fix pattern, or the selector-existence check refused two drafts in a row).
- Tests that hit the 10-attempt local budget without a 3-pass streak.
- Tests with a `ci-local-divergence` verdict — local 3/3 but CI failed; environment-coupled, escalated.
- Tests for which trace artifacts had to be pulled from CI as a fallback (`evidence-from-ci-artifact`).
- Any novel error class that appeared during iteration and is not yet investigated.

## Section rules

- **Headline:** one paragraph, ≤ 4 sentences. Lead with whether the PR is unblocked and whether CI agreed with the local gate.
- **Baseline table:** sort by `failure_rate` desc. Cap at 20 rows; if more, link to a JSON dump alongside the report.
- **Per-test fix log:** include every test in the queue, even `unchanged` and `regressed`. Brevity loses the audit trail. The selector-check row (static / live) is mandatory.
- **Local verification log:** every fix gets a row, including the ones that exhausted the 10-attempt budget. The streak column is the gate's primary signal.
- **CI ratification:** one row per test in the fix queue. If `ci-local-divergence`, include a hypothesis — the report is the handover for the user's next investigation.
- **Guard-rail check:** every line is checked or unchecked — never omitted. An unchecked line forces a manual review.
- **Recommendations:** if none, write `_None._` rather than removing the section.
- **Residual risk:** if none, write `_None._`. Do not hide unresolved items.

## When the run ends in `blocked` status

Write the report regardless.
Replace the "Per-test fix log" verdicts with `blocked — see Residual risk`.
Surface to the user with the residual risk section as the primary handover.

## When the run ends in `ci-local-divergence`

The fix(es) passed the local 3-pass gate but CI disagreed.
Write the report with:

- Per-test fix log entries showing local streak attained, CI verdict regressed / unchanged.
- A `ci-local-divergence` section under residual risk with the hypothesis (env, shared-state, fixture).
- A pointer to the local `.artifacts/<PR_NUMBER>/local/` trace dump and (if pulled) the CI trace.zip.

Do not auto-revert.
The user decides whether to revert, dig into the env divergence, or escalate.

## Optimize-mode section rules

- **Headline:** lead with the total estimated savings number. Skip pleasantries.
- **Top slow tests:** cap at 10 rows. Sort by `p95_duration_ms` desc.
- **Recommendations:** every recommendation has a code block with `before` (measured) and `after` (projected). No code = no recommendation.
- **Estimated savings math:** show the multiplication explicitly — `per-run × occurrence_count = total`. Readers should be able to verify the arithmetic.
- **Risk:** rate every recommendation. A "low-risk 50 ms win" is more valuable than a "high-risk 500 ms win" in most suites.
- **No autonomous edits.** The footer must explicitly say `report-only — no commits made` so a reader cannot mistake this for a stabilize run.
